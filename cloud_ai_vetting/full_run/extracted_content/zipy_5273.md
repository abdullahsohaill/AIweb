## Frequently Asked Questions

### What is session replay?

Session replay shows all activities of a user in a video replay format. Every click and workflow of the users interaction with your website or application is captured and played back. It is not a screen recording. In case of web, it is a DOM replay of the users activity.

### How do you measure Product Analytics?

In Product Analytics one is usually looking for acquisition, activation, retention, referral and revenue metrics. With the entire user journey being captured in Zipy, you can understand these metrics easily.

### What is real-time error monitoring?

Real time error monitoring in an application, is continuously observing the errors in an application and alerting the developers about the issues in the system. As an example, for a web application, if the javascript errors are being continuously being captured as the user is interacting with the application and the developers are being alerted about these errors instantly, it is called frontend error monitoring.

### What kind of monitoring is supported in Zipy?

Zipy supports real time javascript and API error monitoring and performance monitoring along with session replay. If a website or application user encounters an error, it will be captured by Zipy and shown in the Zipy dashboard. These could be any JS error, any 4xx or 5xx error, any console error or any slowness issue.

### What is Zipy solving?

Zipy is solving customer experience problems proactively. This means any glitch that the user faces in the website or web application is captured by Zipy in real time, and the support, success teams, or Product management teams can easily understand the problem and create tickets. Zipy's AI-powered insights help identify and prioritize critical issues, enabling development teams to solve them quickly and efficiently, saving customer churn. Zipy is the one-stop platform for teams to collaborate, whether it's CS, PM, or Dev teams.
