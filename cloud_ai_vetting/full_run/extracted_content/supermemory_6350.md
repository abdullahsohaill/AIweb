## Our vector-graph engine understands users, just like humans do.

Supermemory handles extraction from conversations, PDF, Markdown, Docs, videos, images, and other input formats. Our smart chunking and memory generator work together to build context for truly personalized experiences.
[LINK: How supermemory works →](https://supermemory.ai/docs/how-it-works)
How supermemory works →
#1 - CONNECT

### Plug Supermemory into your stack in minutes.

We built SDKs for everything - OpenAI, Anthropic, AI SDK, Cloudflare, you name it. Installing supermemory takes minutes.
#2 - INGEST

### Bring in any type of data,  from anywhere.

Files, docs, chats, projects, emails, PDFs, app data streams.  Automatic cleaning and chunking for long-form text and structured data.
#3 - EMBED + ENRICH

### Make data smart, not static.

Advanced embeddings for semantic understanding. Graph-based enrichment links related memories across apps and contexts.
#4 - INDEX + STORE

### Memory that scales.

After embedding, memories are indexed into both a vector store and a graph database. This ensures precision and sub-300ms recall.
#5 - Recall

### Retrieve the right memory, instantly.

Sub-300ms recall with semantic + keyword search. Fetch the latest and most relevant memory across sessions, projects, and time.
#6 - EVOLVE

### Memory that evolves with you.

Memories update, extend, derive, and even expire - so your AI stays fresh, relevant, and human-like over time.
