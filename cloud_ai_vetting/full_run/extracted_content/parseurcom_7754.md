## Parseur to the rescue . How does it work ?

Using Parseur is easy. No technical skills required, no complex rules to write, and no need to train an AI model with hundreds of samples.

## 1. Choose what to extract

Upload your first document. Parseur analyzes your document and suggests fields to extract. Review and adjust fields for your use case.

## 2. Automate imports

Parseur supports emails, PDFs (native and scanned), spreadsheets, text files, HTML pages , and more. Our OCR reads many languages and scripts, including handwriting. Upload in the app, set up automatic imports by email, via the API, or through Zapier, Make, or Power Automate.

## 3. Extract data

Pick the engine that fits the document: AI, OCR templates for PDFs, or Text templates for emails and HTML. We recommend starting with the AI engine and using a template engine when strict rules are required.
List the fields you need (invoice number, total, due date, etc.) and Parseur’s AI will extract them for you. Add short instructions when you want extra precision. Because it's layout-aware, it keeps working even when formats change. This is the easiest engine to use and the one we recommend you start with. Prefer strict, rule-based parsing? Use our template engine where determinism matters, both options live in the same workflow.
Create a template by drawing boxes around the data you want to capture. The Zonal OCR engine then extracts those fields from your PDFs automatically. Use Dynamic OCR when text can shift within the layout (for example, totals moving up or down between documents). This engine can also use AI to suggest fields when you create a new template, so setup is faster.
Select the text you want to capture and save it as fields. The template then extracts those fields from every similar message or page. Best for structured or semi-structured emails and HTML where content follows a repeating pattern. Deterministic results with simple rules and no code.
